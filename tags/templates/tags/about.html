Important Note

The data shown for Top Artists Method   was limited to one artist, "Coldplay".
The data shown for Top Tags Method   was taken from Coldplay's top tags, and I personally limited the number of tags for display.

What information was collected?

From the Top Artists Method method, only the artists names were collected, and from the Top Tags Method method, each tag's name and count were collected, as well as the total number of tags.

The reasoning

Before dumping all tags together in one big data set, tags should carry with them some information about the artist they came from. Tags most definately will overlap between artists, but just by being in an artists' tag pool relates all those tags together to an extent.

Now, what?

We are to make some association with one tag to another using the information collected. Before storing each tag, we calculate the relation between every distinct tag pair in an artists' tag pool.

This calculation is based on two factors,
  • A "population" factor based on the number of tags in the artist tag pool
  • A "biased" factor based on each tag's count, i.e. popularity within the artist tag pool

High school students in a class

If we were to calculate some metric between students in a class, the population factor would be an equal relation to every unique pair of students based on the student count, say 1/(total students). However in high school, a student's popularity matters or, whatever. If a cheerleader hangs out with a nerd in the same class, then the cheerleader's popularity might decrease whereas the nerd's popularity might increase. This is just in the movies, of course.

A metric is created for every distinct tag pair in an artist pool before it gets stored in a dataset. Then, the next artist's tags are calculated and stored in the data set. If tags overlap, then the new metric is added to the existing. This process is repeat for all top artists returned to us from Top Artists Method .

Different classes, different students

In high school, there are different classes. When you move from class to class, the students in your class change. Some might be the same, but generally you see new faces in each class. A student might be considered unpopular in Biology class, but highly popular in English. Therefore, his or her relation to every student in every class changes.

last.fm's take

With every search, you can compare the results with last.fm's results for the same tag based on the Get Similar Tags Method method. This method returns the first 50 tags similar to the tag searched, so with this option you will be able to compare only the first 50 tags from the original search.
